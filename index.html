Common Git Bash Cmds
  1. Adding, removing, tracking, ignoring files:
  2. git add . ~adds all changes within directory to staging
  3. git add (file_name) ~adds file to staging
  4. git rm (file_name) ~removes file and stops tracking it
  5. git rm --cached (file_name) ~stops tracking file
  6. git status ~displays changes to files w/in directory
  • Declaring Git Commits:
  • git commit -m "(message)" ~commit changes with a message
  • git commit -am "(message)" ~commit and add all changes to staging
    • >
  • git log ~displays a history of changes
    •
  • Tagging and Pushing Cmds:
  • git tag (tag_name) ~defines a tag to files
  • git push ~update GitHub repo with commits for all existing branches common on the local system and GitHub. Branches and or files on the local system that are new or have not been pushed to the server are not shared
  • git push -all ~push all local branches to GitHub
  • git push (https://remoteURL) (BranchName) ~updates GitHub with all commits for specific branch named
    • >
  • Branch Cmds:
  • git branch ~displays list of all branches
  • git checkout -b (new_branchname) ~Make new branch and switch to it
  • git branch -d (branchname) ~delete branch
  • git merge (branchname) ~merge a different branch with active branch
  • Fetching & Merging:
  • git fetch (https://remoteURL) ~retrieve any commits from GitHub that do not exist locally
  • git merge (https://remoteURL)/BranchName) ~merge remote branch into current working branch
    • *Using fetch before merge allows you to pull files from repo and examine them to resolve conflicts prior to merge.

    Definitions:

    • - Upstream: location of original code (local files)
    • Origin: location of repo (GitHub/remote server to fetch/pull from & push to)
